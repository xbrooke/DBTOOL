## 三、DBTOOL 与车机系统的集成

### 3.1 当前集成点

| 集成点 | 当前实现 | 车机系统 | 状态 |
|--------|--------|--------|------|
| 媒体控制 | AudioManager | MCU 接口 | ⚠️ 不匹配 |
| 音频输出 | 系统默认 | bus-speaker | ✅ 兼容 |
| 应用焦点 | 通知监听 | 应用管理系统 | ⚠️ 不完整 |
| 权限管理 | Android 权限 | OneOSPermissionManager | ⚠️ 不完整 |

### 3.2 需要改进的集成点

#### 1. MCU 接口集成 (优先级: 高)

**当前**: 使用 AudioManager.dispatchMediaKeyEvent()
**问题**: 车机系统可能不支持此方法
**解决**: 实现 MCU 接口支持

**改进方案**:
```java
// 当前实现
AudioManager.dispatchMediaKeyEvent(keyEvent);

// 改进后
if (hasMcuControl()) {
    mcuControl.sendCommand(OPER_ID_PLAY, ...);
} else {
    AudioManager.dispatchMediaKeyEvent(keyEvent);
}
```

#### 2. 应用焦点管理 (优先级: 中)

**当前**: 监听通知
**问题**: 无法获取应用焦点信息
**解决**: 与应用管理系统集成

**改进方案**:
```java
// 监听应用焦点变化
WindowManager.registerWindowManagerCallback(new WindowManagerCallback() {
    @Override
    public void onWindowChange(String packageName) {
        // 根据焦点应用调整行为
    }
});
```

#### 3. 权限管理 (优先级: 中)

**当前**: 标准 Android 权限
**问题**: 车机系统可能有额外权限要求
**解决**: 支持 OneOSPermissionManager

**改进方案**:
```java
// 检查车机系统权限
if (hasOneOSPermission("MCU_CONTROL")) {
    // 使用 MCU 接口
} else {
    // 请求权限
}
```

#### 4. 音频输出管理 (优先级: 低)

**当前**: 依赖系统默认输出
**问题**: 无法控制输出设备
**解决**: 监听音频设备变化

**改进方案**:
```java
// 监听音频设备变化
AudioManager.registerAudioDeviceCallback(new AudioDeviceCallback() {
    @Override
    public void onAudioDevicesAdded(AudioDeviceInfo[] addedDevices) {
        // 处理新设备
    }
});
```
