# DBTOOL 车机平台架构

## 应用定位

**DBTOOL 是安装在车机平板设备上的应用，用于接收和处理来自手机的媒体信息。**

## 核心功能重新定义

### 1. 媒体信息接收
**功能：** 接收来自手机的媒体播放信息
- 歌曲标题
- 艺术家名称
- 专辑名称
- 播放状态
- 专辑封面

**实现方式：**
- ContentProvider (NowPlayingProvider) - 车机系统查询媒体信息
- 广播接收 (DesktopCardReceiver) - 接收手机发送的媒体更新

### 2. 媒体信息存储
**功能：** 保存当前播放的媒体信息
- 使用 MediaNotificationListener.currentMedia 存储
- 线程安全的数据访问
- 支持多应用并发访问

### 3. UI 展示
**功能：** 在车机屏幕上显示媒体信息
- MainActivity - 主界面显示
- 实时更新播放信息
- 显示播放控制按钮

### 4. 控制命令发送
**功能：** 接收用户操作并发送给手机
- 播放/暂停
- 下一曲/上一曲
- 音量调节
- 应用切换

---

## 组件职责重新定义

| 组件 | 原理解 | 正确理解 |
|------|--------|---------|
| **MediaNotificationListener** | 监听手机通知 | ❌ 不适用于车机 |
| **NowPlayingProvider** | 提供媒体信息给车机 | ✅ 车机系统查询媒体信息 |
| **DesktopCardReceiver** | 接收车机控制 | ✅ 接收手机发送的媒体更新 |
| **VehicleCoreService** | 前台服务 | ✅ 保持应用后台运行 |
| **MainActivity** | 显示状态 | ✅ 显示媒体信息和控制 |

## 通信协议

### 手机 → 车机
**协议：** 通过 ContentProvider 或广播

```
手机系统
  ↓
媒体应用发送通知
  ↓
某个中间件或服务
  ↓
车机 DBTOOL 接收
  ├─ 通过 ContentProvider 查询
  └─ 通过广播接收
```

### 车机 → 手机
**协议：** 通过广播或 Intent

```
车机 DBTOOL
  ↓
用户点击控制按钮
  ↓
发送广播或 Intent
  ↓
手机接收
  ↓
媒体应用执行操作
```

## 支持的车机协议

### 1. 亿连协议 (ecarx)
```
广播 Action: ecarx.intent.broadcast.action.MEDIA_CONTROL_RECEIVER
Extra: media_action
  0 = 播放
  1 = 暂停
  2 = 播放/暂停
  3 = 下一曲
  4 = 上一曲
```

### 2. 极氪/几何协议 (geely)
```
广播 Action:
  com.geely.mediawidget.ACTION_WIDGET_TOGGLE_PLAY  → 播放/暂停
  com.geely.mediawidget.ACTION_WIDGET_NEXT         → 下一曲
  com.geely.mediawidget.ACTION_WIDGET_PREV         → 上一曲
```

### 3. 其他车机协议
- 可以通过 DesktopCardReceiver 扩展
- 支持自定义广播 Action
- 支持自定义 Intent 参数

## 关键设计考虑

### 1. 多应用支持
**问题：** 多个手机应用可能同时发送媒体信息
**解决：** 
- 使用优先级机制
- 记录最后更新的应用
- 支持应用切换

### 2. 网络通信
**问题：** 手机和车机可能通过网络连接
**解决：**
- 支持 WiFi 连接
- 支持蓝牙连接
- 支持 USB 连接

### 3. 数据同步
**问题：** 媒体信息可能不同步
**解决：**
- 定期轮询
- 事件驱动更新
- 缓存机制

### 4. 权限管理
**问题：** 车机系统权限不同
**解决：**
- 使用 signature 级别权限
- 支持系统应用权限
- 支持第三方应用权限

## 实际应用场景

### 场景 1：车机显示手机音乐
```
1. 用户在手机上播放网易云音乐
2. 手机通过某种方式通知车机
3. 车机 DBTOOL 接收到媒体信息
4. 车机屏幕显示：
   - 歌曲名称
   - 艺术家
   - 专辑封面
   - 播放进度
   - 控制按钮
```

### 场景 2：车机控制手机音乐
```
1. 用户在车机屏幕上点击"下一曲"按钮
2. DBTOOL 发送控制命令给手机
3. 手机媒体应用接收命令
4. 媒体应用切换到下一首歌
5. 手机通知车机新的媒体信息
6. 车机屏幕更新显示
```

### 场景 3：多应用切换
```
1. 用户在手机上从网易云切换到 QQ 音乐
2. 手机通知车机应用切换
3. DBTOOL 更新显示的媒体信息
4. 车机屏幕显示新应用的媒体信息
```

## 总结

DBTOOL 是一个**车机平板应用**，主要功能是：
1. **接收** 来自手机的媒体信息
